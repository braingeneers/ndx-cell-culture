Validation And DANDI Notes
==========================

NWB files that use this extension require the ``ndx-cell-culture`` namespace
and schema to be available when reading or validating the file.

Recommended Validation Workflow
-------------------------------

Before sharing files, run both PyNWB round-trip checks and NWB Inspector in an
environment where ``ndx-cell-culture`` is installed:

.. code-block:: bash

   pytest -q
   nwbinspector examples/basic_organoid_example.nwb --modules ndx_cell_culture --threshold BEST_PRACTICE_VIOLATION

For project-specific files, replace ``examples/basic_organoid_example.nwb``
with the NWB files generated by your own pipeline.

Recommended-Term Validation
---------------------------

The extension stores recommended vocabulary values as text, following common
NWB practice for evolving community vocabularies. Use the opt-in validator to
find values outside the recommended terms:

.. code-block:: python

   import ndx_cell_culture as ndx

   issues = ndx.validate_recommended_terms(nwbfile)
   for issue in issues:
       print(issue.message)

This validator is intentionally advisory. Non-recommended terms may be valid
when they capture a new preparation type, protocol, reagent, or experimental
practice.

DANDI Guidance
--------------

When depositing data:

* validate NWB files in the same environment used to create them;
* make sure the DANDI dataset documentation names ``ndx-cell-culture`` as an
  extension dependency;
* keep acquisition, stimulus, electrode, imaging, processing, and device
  metadata in core NWB structures where possible;
* use this extension for the cultured-preparation metadata needed to find,
  interpret, and compare the data.
